# Метрики MOEX Select

Метрики помогают Delta team оценивать, насколько сервис упрощает поиск инструментов и насколько стабильно получает данные MOEX ISS.

> Информация не является индивидуальной инвестиционной рекомендацией. Подбор основан на выбранных пользователем параметрах и открытых рыночных данных MOEX ISS.

## Product metrics

| Метрика | Определение | Целевой сигнал |
| --- | --- | --- |
| `time-to-result` | время от открытия формы до получения подборки | сокращение времени поиска |
| `search success rate` | доля подборов, в которых пользователь получил результат и взаимодействовал с карточкой | рост успешных сессий |
| `instrument page CTR` | доля сессий с нажатием `Открыть на MOEX` | переход к изучению инструмента |
| `selection completion rate` | доля начавших форму пользователей, отправивших запрос | понятность анкеты |
| `explanation interaction rate` | доля просмотров карточек с взаимодействием с областью причин при добавлении событий интерфейса | полезность объяснений |
| `repeat usage rate` | доля пользователей, вернувшихся к подбору | повторная ценность сервиса |

## Business metrics

| Метрика | Интерпретация |
| --- | --- |
| Вовлеченность пользователей | рост завершенных подборов и изученных карточек |
| Переходы на страницы инструментов | рост количества переходов из `MOEX Select` на MOEX |
| Торговая активность после изучения | агрегированная оценка связи изучения инструментов с дальнейшей активностью |
| Комиссионный эффект | агрегированное изменение комиссионного потока, связанное с пользовательскими переходами |
| Вовлечение новых инвесторов | доля новых пользователей, успешно завершивших первый подбор |

## Technical metrics

| Метрика | Что измеряется |
| --- | --- |
| API response time | задержка `GET /api/instruments` и `POST /api/recommendations` |
| ISS request success rate | доля успешных обращений `MoexIssClient` |
| Cache hit rate | доля выдач из пяти минутного in-memory cache |
| Reserve catalog usage rate | частота использования встроенного каталога при недоступности набора ISS |
| API error rate | доля запросов, завершившихся ошибкой |

## События интерфейса

| Event | Параметры |
| --- | --- |
| `selection_started` | источник открытия страницы |
| `selection_submitted` | выбранные классы, профиль риска, горизонт |
| `selection_received` | число карточек, время ответа |
| `instrument_opened` | тикер, класс инструмента, позиция в выдаче |
| `complex_instrument_notice_shown` | класс инструмента |

В событиях не требуется сохранять числовые внутренние оценки ранжирования: они служат серверной проверке порядка результата и доступны через диагностический режим API.
